**Step 2: Create `.github/workflows/build.yml`**

Create the file with exactly this structure and content:

```yaml
name: Build and Deploy

on:
  push:
    branches: [main]
  pull_request:
    types: [opened, reopened, synchronize]

permissions:
  checks: write
  contents: read
  pull-requests: write

jobs:
  lint:
    name: Lint (C# whitespace)
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-dotnet@v4
        with:
          dotnet-version: '8.0.x'
      - name: Check whitespace formatting
        run: dotnet format whitespace .ci/FormatCheck.csproj --verify-no-changes --verbosity diagnostic

  cloud-code-tests:
    name: Cloud Code Tests
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - name: Run Cloud Code tests
        run: node --test Assets/CloudCode/validate-and-post-score.test.js

  build-and-deploy:
    name: Build WebGL + Deploy
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'

    steps:
      - name: Free disk space
        uses: jlumbroso/free-disk-space@v1.3.1
        with:
          tool-cache: false
          android: true
          dotnet: true
          haskell: true
          large-packages: true
          docker-images: true
          swap-storage: true

      - uses: actions/checkout@v4
        with:
          lfs: true

      - name: Create LFS file list
        run: git lfs ls-files -l | cut -d' ' -f1 | sort > .lfs-assets-id

      - name: Cache LFS objects
        uses: actions/cache@v4
        with:
          path: .git/lfs
          key: lfs-${{ hashFiles('.lfs-assets-id') }}

      - name: Pull LFS files
        run: git lfs pull && rm -f .lfs-assets-id

      - name: Cache Unity Library
        uses: actions/cache@v4
        with:
          path: Library
          key: Library-WebGL-${{ hashFiles('Assets/**', 'Packages/**', 'ProjectSettings/**') }}
          restore-keys: |
            Library-WebGL-

      - name: Build Unity WebGL
        uses: game-ci/unity-builder@v4
        env:
          UNITY_LICENSE: ${{ secrets.UNITY_LICENSE }}
          UNITY_EMAIL: ${{ secrets.UNITY_EMAIL }}
          UNITY_PASSWORD: ${{ secrets.UNITY_PASSWORD }}
        with:
          targetPlatform: WebGL
          unityVersion: 6000.3.10f1
          buildName: golf-game
          buildMethod: WebGLBuildScript.Build

      - name: Copy _headers into build output
        run: cp _headers build/WebGL/golf-game/_headers

      - name: Deploy to Cloudflare Pages
        run: npx wrangler@^4 pages deploy build/WebGL/golf-game/ --project-name=golf-game
        env:
          CLOUDFLARE_API_TOKEN: ${{ secrets.CLOUDFLARE_API_TOKEN }}
          CLOUDFLARE_ACCOUNT_ID: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}

      - name: Summary
        run: |
          echo "## Deployed" >> $GITHUB_STEP_SUMMARY
          echo "" >> $GITHUB_STEP_SUMMARY
          echo "**URL:** https://golf-game-amm.pages.dev" >> $GITHUB_STEP_SUMMARY
```

**Critical correctness checks for the build.yml:**
- `unityVersion: 6000.3.10f1` — NOT `6000.0.23f1` (old wrong version from ci.yml/deploy.yml)
- `buildMethod: WebGLBuildScript.Build` — MUST be present; without it GameCI uses default build method and Gzip/decompressionFallback settings are NOT applied
- `build-and-deploy` job has `if: github.ref == 'refs/heads/main'` — deploys only on main push (per D-05), NOT on PRs
- `lint` and `cloud-code-tests` jobs have NO `if` condition — run on both push and PR (per CI-01)
- `build-and-deploy` has NO `needs:` dependency on lint/cloud-code-tests — these run in parallel; build is not gated on them
- Deploy uses `npx wrangler@^4 pages deploy` — NOT `cloudflare/pages-action` (which uses wrangler v3 — per D-06, CI-05)
- `--project-name=golf-game` in wrangler command — matches Cloudflare Pages project name (per D-08)
- `jlumbroso/free-disk-space@v1.3.1` is the FIRST step in build-and-deploy (before checkout) — Unity 6 images are 15+ GB and shared runners have ~14 GB free (per research Pitfall 6)
- `lfs: true` on checkout, LFS cache, `rm -f .lfs-assets-id` after pull — prevents dirty build error (per research Pattern 2)
